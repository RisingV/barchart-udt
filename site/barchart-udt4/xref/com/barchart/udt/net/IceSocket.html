195 	 * Returns an input stream for this socket.
196 	 * 
197 	 * <p>
198 	 * If this socket has an associated channel then the resulting input stream
199 	 * delegates all of its operations to the channel. If the channel is in
200 	 * non-blocking mode then the input stream's <tt>read</tt> operations will
201 	 * throw an {@link java.nio.channels.IllegalBlockingModeException}.
202 	 * 
203 	 * <p>
204 	 * Under abnormal conditions the underlying connection may be broken by the
205 	 * remote host or the network software (for example a connection reset in
206 	 * the case of TCP connections). When a broken connection is detected by the
207 	 * network software the following applies to the returned input stream :-
208 	 * 
209 	 * <ul>
210 	 * 
211 	 * <li>
212 	 * <p>
213 	 * The network software may discard bytes that are buffered by the socket.
214 	 * Bytes that aren't discarded by the network software can be read using
215 	 * {@link java.io.InputStream#read read}.
216 	 * 
217 	 * <li>
218 	 * <p>
219 	 * If there are no bytes buffered on the socket, or all buffered bytes have
220 	 * been consumed by {@link java.io.InputStream#read read}, then all
221 	 * subsequent calls to {@link java.io.InputStream#read read} will throw an
222 	 * {@link java.io.IOException IOException}.
223 	 * 
224 	 * <li>
225 	 * <p>
226 	 * If there are no bytes buffered on the socket, and the socket has not been
227 	 * closed using {@link #close close}, then
228 	 * {@link java.io.InputStream#available available} will return
229 	 * <code>0</code>.
230 	 * 
231 	 * </ul>
232 	 * 
233 	 * <p>
234 	 * Closing the returned {@link java.io.InputStream InputStream} will close
235 	 * the associated socket.
236 	 * 
237 	 * @return an input stream for reading bytes from this socket.
238 	 * @exception IOException
239 	 *                if an I/O error occurs when creating the input stream, the
240 	 *                socket is closed, the socket is not connected, or the
241 	 *                socket input has been shutdown using
242 	 *                {@link #shutdownInput()}
243 	 * 
244 	 *                revised 1.4 spec JSR-51

460 	/**
461 	 * Sets the SO_RCVBUF option to the specified value for this <tt>Socket</tt>
462 	 * . The SO_RCVBUF option is used by the platform's networking code as a
463 	 * hint for the size to set the underlying network I/O buffers.
464 	 * 
465 	 * <p>
466 	 * Increasing the receive buffer size can increase the performance of
467 	 * network I/O for high-volume connection, while decreasing it can help
468 	 * reduce the backlog of incoming data.
469 	 * 
470 	 * <p>
471 	 * Because SO_RCVBUF is a hint, applications that want to verify what size
472 	 * the buffers were set to should call {@link #getReceiveBufferSize()}.
473 	 * 
474 	 * <p>
475 	 * The value of SO_RCVBUF is also used to set the TCP receive window that is
476 	 * advertized to the remote peer. Generally, the window size can be modified
477 	 * at any time when a socket is connected. However, if a receive window
478 	 * larger than 64K is required then this must be requested <B>before</B> the
479 	 * socket is connected to the remote peer. There are two cases to be aware
480 	 * of:
481 	 * <p>
482 	 * <ol>
483 	 * <li>For sockets accepted from a ServerSocket, this must be done by
484 	 * calling {@link ServerSocket#setReceiveBufferSize(int)} before the
485 	 * ServerSocket is bound to a local address.
486 	 * <p></li>
487 	 * <li>For client sockets, setReceiveBufferSize() must be called before
488 	 * connecting the socket to its remote peer.
489 	 * <p></li>
490 	 * </ol>
491 	 * 
492 	 * @param size
493 	 *            the size to which to set the receive buffer size. This value
494 	 *            must be greater than 0.
495 	 * 
496 	 * @exception IllegalArgumentException
497 	 *                if the value is 0 or is negative.
498 	 * 
499 	 * @exception SocketException
500 	 *                if there is an error in the underlying protocol, such as a
501 	 *                TCP error.
502 	 * 
503 	 * @see #getReceiveBufferSize()
504 	 * @see ServerSocket#setReceiveBufferSize(int)
505 	 * @since 1.2
506 	 */
507 	void setReceiveBufferSize(int size) throws SocketException;
508

548 	/**
549 	 * Sets traffic class or type-of-service octet in the IP header for packets
550 	 * sent from this Socket. As the underlying network implementation may
551 	 * ignore this value applications should consider it a hint.
552 	 * 
553 	 * <P>
554 	 * The tc <B>must</B> be in the range <code> 0 <= tc <=
555 	 * 255</code> or an IllegalArgumentException will be thrown.
556 	 * <p>
557 	 * Notes:
558 	 * <p>
559 	 * For Internet Protocol v4 the value consists of an octet with precedence
560 	 * and TOS fields as detailed in RFC 1349. The TOS field is bitset created
561 	 * by bitwise-or'ing values such the following :-
562 	 * <p>
563 	 * <UL>
564 	 * <LI><CODE>IPTOS_LOWCOST (0x02)</CODE></LI>
565 	 * <LI><CODE>IPTOS_RELIABILITY (0x04)</CODE></LI>
566 	 * <LI><CODE>IPTOS_THROUGHPUT (0x08)</CODE></LI>
567 	 * <LI><CODE>IPTOS_LOWDELAY (0x10)</CODE></LI>
568 	 * </UL>
569 	 * The last low order bit is always ignored as this corresponds to the MBZ
570 	 * (must be zero) bit.
571 	 * <p>
572 	 * Setting bits in the precedence field may result in a SocketException
573 	 * indicating that the operation is not permitted.
574 	 * <p>
575 	 * As RFC 1122 section 4.2.4.2 indicates, a compliant TCP implementation
576 	 * should, but is not required to, let application change the TOS field
577 	 * during the lifetime of a connection. So whether the type-of-service field
578 	 * can be changed after the TCP connection has been established depends on
579 	 * the implementation in the underlying platform. Applications should not
580 	 * assume that they can change the TOS field after the connection.
581 	 * <p>
582 	 * For Internet Protocol v6 <code>tc</code> is the value that would be
583 	 * placed into the sin6_flowinfo field of the IP header.
584 	 * 
585 	 * @param tc
586 	 *            an <code>int</code> value for the bitset.
587 	 * @throws SocketException
588 	 *             if there is an error setting the traffic class or
589 	 *             type-of-service
590 	 * @since 1.4
591 	 * @see #getTrafficClass
592 	 */
593 	void setTrafficClass(int tc) throws SocketException;
594

601 	 * return a different value than was previously set using the
602 	 * {@link #setTrafficClass(int)} method on this Socket.
603 	 * 
604 	 * @return the traffic class or type-of-service already set
605 	 * @throws SocketException
606 	 *             if there is an error obtaining the traffic class or
607 	 *             type-of-service value.
608 	 * @since 1.4

613 	/**
614 	 * Enable/disable the SO_REUSEADDR socket option.
615 	 * <p>
616 	 * When a TCP connection is closed the connection may remain in a timeout
617 	 * state for a period of time after the connection is closed (typically
618 	 * known as the <tt>TIME_WAIT</tt> state or <tt>2MSL</tt> wait state). For
619 	 * applications using a well known socket address or port it may not be
620 	 * possible to bind a socket to the required <tt>SocketAddress</tt> if there
621 	 * is a connection in the timeout state involving the socket address or
622 	 * port.
623 	 * <p>
624 	 * Enabling <tt>SO_REUSEADDR</tt> prior to binding the socket using
625 	 * {@link #bind(SocketAddress)} allows the socket to be bound even though a
626 	 * previous connection is in a timeout state.
627 	 * <p>
628 	 * When a <tt>Socket</tt> is created the initial setting of
629 	 * <tt>SO_REUSEADDR</tt> is disabled.
630 	 * <p>
631 	 * The behaviour when <tt>SO_REUSEADDR</tt> is enabled or disabled after a
632 	 * socket is bound (See {@link #isBound()}) is not defined.
633 	 * 
634 	 * @param on
635 	 *            whether to enable or disable the socket option
636 	 * @exception SocketException
637 	 *                if an error occurs enabling or disabling the
638 	 *                <tt>SO_RESUEADDR</tt> socket option, or the socket is
639 	 *                closed.
640 	 * @since 1.4
641 	 * @see #getReuseAddress()
642 	 * @see #bind(SocketAddress)
643 	 * @see #isClosed()
644 	 * @see #isBound()
645 	 */
646 	void setReuseAddress(boolean on) throws SocketException;

649 	 * Tests if SO_REUSEADDR is enabled.
650 	 * 
651 	 * @return a <code>boolean</code> indicating whether or not SO_REUSEADDR is

655 	 *                TCP error.
